Robotic Arm Project Details

Project Summary — Vision-Guided Robotic Arm

A compact, 4-DOF desktop robotic arm that tracks a visual target in real time. It marries an ESP32 servo controller with an ESP32-CAM and a small Python vision app to keep the camera centered on a printed ArUco marker.

  • 4 servos: base, shoulder, elbow, and a tilt joint carrying the camera
  • ESP32 control (PlatformIO/C++), with smooth motion and joint-limit safety (elbow window coupled to shoulder angle)
  • ESP32-CAM provides an MJPEG stream (320×240) resolution for low-latency tracking
  • OpenCV (Python) detects a 5×5 ArUco marker and sends joint targets over USB serial using a simple <base> <shoulder> <elbow> <tilt> command
  • Visual servoing: per-frame corrections with deadband and rate limits to avoid jitter
  • Home/presets: on-board preferences remember the last pose; configurable “home” including tilt
  • 3D-printed structure designed in SolidWorks; lightweight and easy to iterate
  • Modular code: swap vision sources or control modes (manual serial, scripted moves) without rewiring
  • Future add-ons: gripper, PID tuning, higher-res streaming, camera/arm calibration for 3D tasks

Circuit description

  • The ESP32 controls 4x SG90 servos (base, shoulder, elbow, wrist, gripper) using PWM.
  • An ESP32-CAM provides live video; the arm can track 5×5 ArUco markers from the camera feed.
  • Servos are powered from a dedicated 5 V rail with bulk/decoupling capacitors; logic power is isolated but shares a common ground.
  • Optional limit switches on joints enable a homing routine; software limits prevent over-travel.
  • A simple USB serial protocol supports live tuning (offsets/speeds) and streaming telemetry.
